# 02 — Actors & Use Cases

---

## System Rules (ข้อตกลงหลักของระบบ)

| กฎ | รายละเอียด |
|---|---|
| **Approval Required** | การจองทุกครั้งต้องได้รับการอนุมัติจาก Approver เสมอ ไม่มี auto-approve |
| **Approver Scope** | Approver สามารถอนุมัติได้ทุกห้องในระบบ ไม่จำกัดห้องหรือหน่วยงาน |
| **Member Activation** | สมาชิกใหม่ต้องรอ Admin อนุมัติก่อน จึงจะ Login เข้าระบบได้ |

## Actors Overview

```
┌─────────────────────────────────────────────────────────────┐
│                      P6BookingMe                            │
│                                                             │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐             │
│   │  Member  │    │ Approver │    │  Admin   │             │
│   │(พนักงาน) │    │          │    │          │             │
│   └──────────┘    └──────────┘    └──────────┘             │
│        │               │               │                   │
│        └───────────────┴───────────────┘                   │
│                  inherits ↑                                 │
│           Member ⊂ Approver ⊂ Admin (privilege)            │
└─────────────────────────────────────────────────────────────┘
```

| Actor | คำอธิบาย | เงื่อนไขการเข้าใช้งาน |
|---|---|---|
| **Member** | พนักงานขององค์กรที่สมัครสมาชิกและได้รับการอนุมัติจาก Admin แล้ว | Admin อนุมัติ account |
| **Approver** | สมาชิกที่ถูก Admin กำหนดสิทธิ์เพิ่มเติมให้อนุมัติการจองได้ | Admin assign role |
| **Admin** | ผู้ดูแลระบบสูงสุด จัดการทุกส่วนของระบบ | สร้างโดย system / กำหนดใน config |

## Booking Status State Machine

```
                    ┌─────────┐
         สมัครสมาชิก│ PENDING │ รอ Admin อนุมัติ account
                    └────┬────┘
              Admin       │        Admin
             Approve      │        Reject
                    ┌─────▼────┐  ┌──────────┐
                    │ ACTIVE   │  │ REJECTED │ (ไม่สามารถ login ได้)
                    └──────────┘  └──────────┘

─────────────────────────────────────────────────────

                    ┌─────────┐
          Member จอง│ PENDING │ รอ Approver อนุมัติ
                    └────┬────┘
          Approver        │        Approver
          Approve         │        Reject
                    ┌─────▼────┐  ┌──────────┐
                    │CONFIRMED │  │ REJECTED │
                    └────┬─────┘  └──────────┘
                         │
               Member/Admin ยกเลิก
                    ┌─────▼────┐
                    │CANCELLED │
                    └──────────┘
```

## Actor 1: Member

> พนักงานขององค์กรที่สมัครสมาชิกและ Admin อนุมัติแล้ว

### Use Cases

| ID | Use Case | คำอธิบาย | Pre-condition | Post-condition |
|---|---|---|---|---|
| M01 | สมัครสมาชิก | กรอกข้อมูลส่วนตัว (ชื่อ, อีเมล, หน่วยงาน, รหัสพนักงาน) เพื่อขอเป็นสมาชิก | ยังไม่มี account | Account อยู่ในสถานะ Pending รอ Admin อนุมัติ |
| M02 | เข้าสู่ระบบ | Login ด้วย email + password | Account ได้รับการ Approve จาก Admin แล้ว | ได้รับ JWT Token เข้าใช้ระบบ |
| M03 | ออกจากระบบ | Logout ล้าง session | Login อยู่ | Session ถูกยกเลิก |
| M04 | แก้ไขข้อมูลส่วนตัว | แก้ไขชื่อ, เบอร์โทร, รูปโปรไฟล์ (ไม่สามารถแก้ email และรหัสพนักงาน) | Login อยู่ | ข้อมูลส่วนตัวถูกอัปเดต |
| M05 | ค้นหาห้องประชุม | กรองห้องตาม วันที่, ช่วงเวลา, ความจุ, สถานที่/อาคาร | Login อยู่ | แสดงรายการห้องที่ว่างตาม criteria |
| M06 | ดูรายละเอียดห้องประชุม | ดูข้อมูลห้อง เช่น ขนาด, อุปกรณ์, รูปภาพ, ตารางการจองของห้อง | Login อยู่ | แสดงข้อมูลห้องและ time slots ที่ว่าง |
| M07 | จองห้องประชุม | เลือกห้อง, วันที่, เวลาเริ่ม-สิ้นสุด, หัวข้อการประชุม, จำนวนผู้เข้าร่วม | Login อยู่, ช่วงเวลานั้นห้องว่าง | Booking ถูกสร้างในสถานะ **Pending** |
| M08 | ดูรายการจองของตัวเอง | ดูประวัติการจองทั้งหมด พร้อมสถานะ (Pending/Confirmed/Rejected/Cancelled) | Login อยู่ | แสดงรายการจองพร้อมสถานะ |
| M09 | ยกเลิกการจอง | ยกเลิกการจองที่ยังอยู่ในสถานะ Pending หรือ Confirmed | Login อยู่, เป็นเจ้าของ booking | Booking เปลี่ยนสถานะเป็น Cancelled |
| M10 | ดูปฏิทินการจองภาพรวม | ดูตาราง Calendar แสดงการจองของทุกคนในระบบ (เพื่อวางแผน) | Login อยู่ | แสดง Calendar พร้อม time slots ที่ถูกจอง |
| M11 | รับการแจ้งเตือน | รับ notification เมื่อสถานะการจองเปลี่ยน (Approved / Rejected) | มี booking อยู่ | แสดง notification ในระบบ |
| M12 | แก้ไขการจอง | แก้ไขรายละเอียด (วาระ, วันเวลา, จำนวนคน) ของรายการที่จองไว้ | Login อยู่, เป็นเจ้าของ booking | Booking ถูกอัปเดต (หาก Confirmed แล้วจะกลับไป Pending) |
| M13 | จองซ้ำด่วน (Duplicate) | สร้างรายการจองใหม่โดยคัดลอกข้อมูลจากรายการเดิม (Template) | Login อยู่, มีประวัติการจอง | เปิดหน้าจองห้องพร้อมกรอกข้อมูลเดิมให้อัตโนมัติ |

## Actor 2: Approver

> สมาชิกที่ถูก Admin กำหนดให้มีสิทธิ์อนุมัติการจองห้องประชุม สามารถอนุมัติได้ทุกห้องในระบบ

### Inherits
Approver ทำทุกอย่างที่ **Member** ทำได้ (M01–M11)

### Use Cases เพิ่มเติม

| ID | Use Case | คำอธิบาย | Pre-condition | Post-condition |
|---|---|---|---|---|
| AP01 | ดูรายการจองที่รอการอนุมัติ | ดูรายการ Booking ทั้งหมดที่อยู่ในสถานะ Pending | Login อยู่, มี role Approver | แสดงรายการ Booking Pending พร้อมรายละเอียด |
| AP02 | อนุมัติการจอง | กด Approve บน Booking ที่ Pending | Login อยู่, Booking อยู่ในสถานะ Pending | Booking เปลี่ยนเป็น **Confirmed**, Member ได้รับแจ้งเตือน |
| AP03 | ปฏิเสธการจอง | กด Reject พร้อมระบุเหตุผล | Login อยู่, Booking อยู่ในสถานะ Pending | Booking เปลี่ยนเป็น **Rejected**, Member ได้รับแจ้งเตือนพร้อมเหตุผล |
| AP04 | ดูประวัติการอนุมัติ | ดูรายการ Booking ที่ตัวเองเคย Approve/Reject พร้อม timestamp | Login อยู่ | แสดงประวัติการอนุมัติ |

## Actor 3: Admin

> ผู้ดูแลระบบสูงสุด มีสิทธิ์เข้าถึงทุกส่วนของระบบ

### Inherits
Admin ทำทุกอย่างที่ **Member** และ **Approver** ทำได้

### Use Cases เพิ่มเติม

| ID | Use Case | คำอธิบาย | Pre-condition | Post-condition |
|---|---|---|---|---|
| AD01 | ดูรายการสมาชิกที่รอการอนุมัติ | ดูรายชื่อผู้สมัครที่ยังอยู่ในสถานะ Pending | Login อยู่, มี role Admin | แสดงรายการ Pending Members |
| AD02 | อนุมัติการสมัครสมาชิก | Approve account ของพนักงานที่สมัครใหม่ | Member อยู่ในสถานะ Pending | Account เปลี่ยนเป็น Active, Member Login ได้ |
| AD03 | ปฏิเสธการสมัครสมาชิก | Reject account พร้อมระบุเหตุผล | Member อยู่ในสถานะ Pending | Account ถูก Reject, แจ้งเตือนผู้สมัคร |
| AD04 | จัดการสมาชิก | ดู, ระงับ (Suspend), หรือ Reactivate account ของสมาชิก | Login อยู่ | สถานะสมาชิกถูกอัปเดต |
| AD05 | กำหนด Approver | เพิ่มหรือถอด role Approver ให้กับสมาชิก | สมาชิกเป็น Active | Role ของสมาชิกถูกอัปเดต |
| AD06 | เพิ่มห้องประชุม | สร้างห้องใหม่พร้อมกำหนด ชื่อ, ความจุ, อุปกรณ์, รูปภาพ, สถานที่ | Login อยู่ | ห้องใหม่ถูกสร้างในระบบ |
| AD07 | แก้ไขข้อมูลห้องประชุม | อัปเดตรายละเอียดห้อง | ห้องมีอยู่ในระบบ | ข้อมูลห้องถูกอัปเดต |
| AD08 | ปิด/เปิดการใช้งานห้อง | ปิดห้องชั่วคราว (เช่น ซ่อมแซม) โดยไม่ลบออกจากระบบ | ห้องมีอยู่ในระบบ | ห้องเปลี่ยนสถานะ Active/Inactive |
| AD09 | ตั้งค่ากฎการจอง | กำหนด: จองล่วงหน้าได้กี่วัน, จองต่อครั้งได้กี่ชั่วโมง, จองได้กี่ครั้งต่อวันต่อคน | Login อยู่ | กฎถูกบันทึกและมีผลทันที โดยไม่ต้อง deploy |
| AD10 | ดู Dashboard | ดูสถิติภาพรวม: อัตราการใช้ห้อง, จำนวนการจองรายวัน/สัปดาห์, ห้องที่ถูกจองบ่อย | Login อยู่ | แสดง Dashboard พร้อมกราฟและตัวเลขสำคัญ |
| AD11 | ดูรายการจองทั้งหมด | ดู Booking ทุกรายการของทุก Member ในระบบ พร้อม filter | Login อยู่ | แสดงรายการ Booking ทั้งหมด |
| AD12 | ยกเลิกการจองใดก็ได้ | ยกเลิก Booking ของ Member คนใดก็ได้ พร้อมระบุเหตุผล | Booking อยู่ในสถานะ Pending หรือ Confirmed | Booking เปลี่ยนเป็น Cancelled, แจ้งเตือน Member |
| AD13 | ดู Audit Log | ดูประวัติทุกการกระทำสำคัญในระบบ (Who, When, What, Why) | Login อยู่ | แสดง Audit Log พร้อม filter ตามวันที่/ผู้ใช้/ประเภทกิจกรรม |
